## 3. Agent Definitions (Phase 5 Scope)

**What stays in Python** (`scripts/analyst.py`):

- `collect` subcommand: API calls to X, SQLite writes via `db_manager.py`
- `summary` subcommand: SQL queries, raw JSON output to `data/metrics_{date}_{account}.json`
- `import` subcommand: CSV/JSON import to SQLite

**What moves to Claude** (new section in `agents/analyst.md`):

| Capability | Current Owner | New Owner | Why Move |
|---|---|---|---|
| Anomaly detection (followers >10% change) | Marc (Step P8) | Analyst Claude subagent | Analyst has all metrics context; removes duplicated logic from Marc |
| Category performance breakdown | Not done | Analyst Claude subagent | Data exists in content plan + metrics; never computed |
| A/B test evaluation | Not done | Analyst Claude subagent | Strategy defines test; metrics have results; no one connects them |
| Outbound effectiveness summary | Not done | Analyst Claude subagent | Outbound log has data; never analyzed |
| Trend comparison (today vs yesterday) | Not done | Analyst Claude subagent | SQLite has history; only today's data used |
| Daily report composition | Marc (Step P8 manual) | Analyst Claude subagent | Telegram message composed with full context |
| Follower growth rate | Not done | Analyst Claude subagent | Raw delta exists; percentage never computed |

**New output**: `data/daily_report_{YYYYMMDD}.json` — see Section 6.1

**Execution flow**:

```
Marc (Step P8) spawns Analyst subagent
  → Analyst reads agents/analyst.md "Intelligence Mode"
  → Reads data/metrics_{date}_EN.json + data/metrics_{date}_JP.json
  → Reads data/pipeline_state_{date}.json (for War Room scores)
  → Reads data/outbound_log_{date}.json
  → Reads data/daily_report_{yesterday}.json (if exists, for trend comparison)
  → Reasons about all data: anomalies, categories, A/B tests, trends
  → Writes data/daily_report_{date}.json
Marc reads daily_report → sends telegram_report via telegram_send.py
Marc sends each telegram_alerts entry separately
```

### 3.2 Scout — Daily Intelligence Mode

**What stays in Python** (`scripts/scout.py`):

- API collection for all competitors (unchanged)
- Keyword search (unchanged)
- User ID resolution (unchanged)
- Basic metric calculation (unchanged)
- **NEW**: Pre-compaction statistics computed in Python before stripping data — reply contamination rate, impression-based engagement rates, dynamic trending threshold. These are included in the compact output as a `_pre_analysis` section so Claude doesn't need access to the stripped `recent_posts` arrays.

**What moves to Claude** (new section in `agents/scout.md`):

| Capability | Problem in Current Python | Claude Solution |
|---|---|---|
| Reply filtering | 36.9% of sampled tweets are @reply tweets that distort engagement rates | Python computes `reply_contamination_rate` and `engagement_rate_excl_replies` per competitor pre-compaction; Claude interprets the results, flags problematic accounts, writes narrative |
| Impression-based engagement | `impression_count` returned by API but never used | Python computes `impression_based_engagement` per competitor pre-compaction; Claude compares to follower-based rates, identifies discrepancies |
| Trending threshold | Hardcoded `like_count >= 100` returns empty (Basic plan tweets too recent) | Python computes dynamic threshold (mean + 2*stddev) and identifies trending posts pre-compaction; Claude curates the list and explains significance |
| New account filtering | 59 raw accounts, no quality filter | Claude filters: min 500 followers, account age >30 days, non-zero engagement. Raw account data is small enough to include in compact output. |
| Hashtag signal | 92.7% of competitors use zero hashtags — empty table output | Claude notes absence as strategic signal ("top performers don't rely on hashtags") |
| Executive summary | No narrative output — only raw JSON | Claude writes 3-5 bullet executive summary per market with specific numbers |

**New flags**:

- `--raw`: Writes full report to `data/scout_raw_{YYYYMMDD}.json` (same schema as current report, different filename). Always writes the full file regardless of `--compact`.
- `--compact`: Additionally writes a compact version to `data/scout_compact_{YYYYMMDD}.json` — strips `recent_posts` arrays but includes a `_pre_analysis` section with statistics computed from the full data before stripping. Reduces ~457KB to ~30KB for Claude context window.

When both flags are used together: Python writes **two files** — the full raw file (for the enriched report) and the compact file (for Claude's analysis input).

**Context window constraint**: Current scout report is 457KB (~120K tokens). The compact version drops to ~30KB (~8K tokens), well within Claude's context. Pre-compaction statistics ensure no analytical data is lost.

**Execution flow**:

```
Marc (Step 2) spawns Scout subagent
  → Scout reads agents/scout.md "Daily Intelligence Mode"
  → Runs: python3 scripts/scout.py --raw --compact
      Python writes data/scout_raw_{date}.json (full, ~457KB)
      Python writes data/scout_compact_{date}.json (~30KB, with _pre_analysis)
  → Reads data/scout_compact_{date}.json (compact version, ~30KB)
  → Analyzes using _pre_analysis stats + compact data
  → Reads data/scout_raw_{date}.json (full file) as base for enriched output
  → Writes data/scout_report_{date}.json (full raw data + analysis section)
Strategist consumes scout_report as usual (backward compatible)
```

### 3.3 Publisher — Smart Outbound Mode

**What stays in Python** (`scripts/publisher.py`):

- `post` subcommand: unchanged (safety-critical, human-gated)
- Rate limiting: unchanged (Python enforces hard limits)
- Media upload: unchanged
- Random delays between API calls: unchanged (30-120s)

**What moves to Claude** (new section in `agents/publisher.md`):

| Capability | Current Behavior | Claude Solution |
|---|---|---|
| Tweet selection for likes | Likes all recent tweets blindly | Select tweets relevant to AI beauty niche; skip personal/off-topic |
| Reply target selection | Always `recent_tweets[0]` | Pick most relevant/engaging tweet based on content |
| Reply crafting | `random.choice(reply_templates)` | Contextual reply referencing actual tweet content, using templates as style guidance |
| Relevance filtering | None — engages with all target accounts equally | Skip targets whose recent content is off-topic or controversial |
| Failure classification | All errors logged identically | (Phase 5 scope: plan only, not execution triage) |

**New helper**: `scripts/publisher_outbound_data.py`

- Fetches target account info + recent tweets
- Output: JSON with each target's handle, followers, bio, recent tweet texts + metrics
- ~120 lines, uses existing `XApiClient`

**New subcommand**: `publisher.py smart-outbound --account EN --plan data/outbound_plan_{date}_EN.json`

- Reads Claude-generated plan, executes each action (like, reply, follow) with same rate limits and delays as current `outbound`

**New output**: `data/outbound_plan_{YYYYMMDD}_{account}.json` — see Section 6.4

**Approval policy**: Required initially (Marc sends plan preview to Telegram, waits for approval). After 1-2 weeks of verified quality, switch to autonomous.

**Execution flow**:

```
Marc (Step P4) spawns Publisher subagent
  → Publisher reads agents/publisher.md "Smart Outbound Mode"
  → Reads data/strategy_{date}.json (outbound_strategy section)
  → Reads data/content_plan_{date}_{account}.json (reply_templates)
  → Runs: python3 scripts/publisher_outbound_data.py --account EN --targets "@target1,@target2"
  → Reads target data JSON
  → For each target: selects tweets to like, picks best reply target, crafts contextual reply
  → Writes data/outbound_plan_{date}_{account}.json
Marc validates plan → (initially) sends preview to Telegram for approval
After approval: python3 scripts/publisher.py smart-outbound --account EN --plan data/outbound_plan_{date}_EN.json
```

**Fallback**: If Claude subagent fails, Marc falls back to legacy `publisher.py outbound --account EN` (context-blind but functional).

## 4. Implementation Architecture

### 4.1 "Claude Brain, Python Hands" Pattern

| Agent | Claude Does (Reasoning) | Python Does (Execution) |
|---|---|---|
| **Analyst** | Anomaly detection, category breakdown, A/B evaluation, trend analysis, report composition | API calls, SQLite writes, raw summary generation |
| **Scout** | Reply filtering, impression engagement, trending thresholds, account filtering, executive summary | API collection, keyword search, user ID resolution, basic metrics |
| **Publisher** | Tweet selection, contextual reply crafting, relevance filtering | API calls (like, reply, follow), rate limiting, delays, media upload |

### 4.2 Execution Model

Sequence diagram for each converted agent:

```
Analyst Intelligence:
  Marc ──▶ claude -p "Analyst Intelligence Mode..."
    Analyst reads metrics_EN.json, metrics_JP.json
```

### 4.4 Harness Principles Applied

| Principle | Layer | Phase 5 Implication |
|---|---|---|
| **H1**: Shell catches crashes | `run_pipeline.sh` / `run_task.sh` | Unchanged — shell wrapper catches subagent process failures |

### 5.4 `scripts/scout.py` — `--raw` and `--compact` Flags

Python pseudo-implementation:

```python
def parse_args():
    """Extended argument parser with --raw and --compact flags."""
    parser = argparse.ArgumentParser(description="Scout agent — competitor data collection")
    # ... existing args ...
    parser.add_argument("--raw", action="store_true",
                        help="Write to scout_raw_{date}.json instead of scout_report_{date}.json")
    parser.add_argument("--compact", action="store_true",
                        help="Additionally write scout_compact_{date}.json with pre-analysis stats")
    return parser.parse_args()

def compute_pre_analysis(report: dict) -> dict:
    """Compute statistics from full data before compaction.

    Analyzes the full recent_posts arrays to produce pre-compaction
    statistics that Claude can use without needing the raw tweet data.
    This solves the problem of Claude needing analytical results from
    data too large to fit in its context window.

    Args:
        report: Full scout report dict with all recent_posts

    Returns:
        Dict with reply_contamination, impression_engagement,
        trending, and hashtag_usage statistics
    """
    all_tweets = []
    per_competitor = []

    for comp in report.get("competitors", []):
        posts = comp.get("recent_posts", [])
        reply_count = sum(1 for p in posts if p.get("text", "").startswith("@"))
        total = len(posts)

        # Reply-filtered engagement rate
        non_reply_posts = [p for p in posts if not p.get("text", "").startswith("@")]
        followers = comp.get("followers", 0)
        if non_reply_posts and followers > 0:
            eng_sum = sum(
                (p.get("like_count", 0) + p.get("retweet_count", 0) +
                 p.get("reply_count", 0) + p.get("quote_count", 0))
                for p in non_reply_posts
            )
            eng_rate_excl_replies = eng_sum / (len(non_reply_posts) * followers)
        else:
            eng_rate_excl_replies = 0.0

        # Impression-based engagement (where available)
        imp_posts = [p for p in posts if p.get("impression_count", 0) > 0]
        if imp_posts:
            imp_eng = sum(
                (p.get("like_count", 0) + p.get("retweet_count", 0) +
                 p.get("reply_count", 0) + p.get("quote_count", 0))
                / p["impression_count"]
                for p in imp_posts
            ) / len(imp_posts)
        else:
            imp_eng = None

        per_competitor.append({
            "handle": comp.get("handle"),
            "market": comp.get("market"),
            "reply_rate": reply_count / total if total > 0 else 0.0,
            "reply_count": reply_count,
            "total_sampled": total,
            "engagement_rate_excl_replies": round(eng_rate_excl_replies, 6),
            "impression_based_rate": round(imp_eng, 6) if imp_eng else None,
        })

        all_tweets.extend(posts)

    # Dynamic trending threshold
    all_likes = [t.get("like_count", 0) for t in all_tweets]
    if all_likes:
        mean_likes = sum(all_likes) / len(all_likes)
        variance = sum((x - mean_likes) ** 2 for x in all_likes) / len(all_likes)
        stddev_likes = variance ** 0.5
        trending_threshold = mean_likes + 2 * stddev_likes
    else:
        trending_threshold = 0

    trending_posts = [
        {"tweet_id": t.get("id"), "handle": t.get("author_handle", ""),
         "likes": t.get("like_count", 0), "text_preview": t.get("text", "")[:80]}
        for t in all_tweets if t.get("like_count", 0) >= trending_threshold
    ]

    # Overall contamination
    total_replies = sum(c["reply_count"] for c in per_competitor)
    total_sampled = sum(c["total_sampled"] for c in per_competitor)

    # Hashtag usage
    comps_with_zero = sum(1 for c in report.get("competitors", [])
                          if len(c.get("hashtags_used", {})) == 0)

    return {
        "reply_contamination": {
            "overall_rate": round(total_replies / total_sampled, 3) if total_sampled > 0 else 0,
            "total_replies": total_replies,
            "total_sampled": total_sampled,
            "per_competitor": per_competitor,
        },
        "impression_engagement": {
            "per_competitor": [
                {"handle": c["handle"], "rate": c["impression_based_rate"]}
                for c in per_competitor if c["impression_based_rate"] is not None
            ]
        },
        "trending": {
            "threshold": round(trending_threshold, 1),
            "method": "mean + 2*stddev of all sampled tweet likes",
            "posts": trending_posts[:20],  # cap at 20
        },
        "hashtag_usage": {
            "competitors_with_zero": comps_with_zero,
            "competitors_with_zero_pct": round(
                comps_with_zero / len(report.get("competitors", [1])) * 100, 1),
        }
    }

def compact_report(report: dict) -> dict:
    """Strip large arrays and add pre-analysis stats for Claude.

    1. Computes pre-analysis statistics from full recent_posts data
    2. Strips recent_posts arrays (~10 tweets per competitor)
    3. Strips full tweet text from trending_posts
    4. Trims new_accounts_discovered to summary fields
    5. Adds _pre_analysis section with computed stats

    Reduces ~457KB to ~30KB for Claude context window compatibility.

    Args:
        report: Full scout report dict

    Returns:
        Compact report dict with _pre_analysis section
    """
    # Compute stats BEFORE stripping data
    pre_analysis = compute_pre_analysis(report)

    compact = copy.deepcopy(report)
    for competitor in compact.get("competitors", []):
        competitor.pop("recent_posts", None)
    for topic in compact.get("trending_posts", []):
        topic.pop("full_text", None)
    # Trim new_accounts_discovered to essential fields
    compact["new_accounts_discovered"] = [
        {k: v for k, v in acc.items()
         if k in ("handle", "followers", "engagement_rate", "source", "tweet_count")}
        for acc in compact.get("new_accounts_discovered", [])
    ]

    compact["_pre_analysis"] = pre_analysis
    return compact

def run(self):
    """Main execution flow."""
    # ... existing collection logic ...

    report = self.build_report()

    # --raw: always write full file to scout_raw_{date}.json
    if self.args.raw:
        raw_path = f"data/scout_raw_{self.date}.json"
        self.write_json(raw_path, report)

    # --compact: additionally write compact file to scout_compact_{date}.json
    if self.args.compact:
        compact = compact_report(report)
        compact_path = f"data/scout_compact_{self.date}.json"
        self.write_json(compact_path, compact)

    # Default (no flags): write to scout_report_{date}.json as before
    if not self.args.raw:
        output_path = f"data/scout_report_{self.date}.json"
        self.write_json(output_path, report)
```

```python
def run_smart_outbound(self, account: str, plan_path: str, dry_run: bool) -> int:
    """Execute a Claude-generated outbound engagement plan.

    Reads a pre-computed plan JSON and executes each action
    (like, reply, follow) with the same rate limiting and delays
    as the legacy outbound command.

    Args:
        account: "EN" or "JP"
        plan_path: Path to outbound_plan_{date}_{account}.json
        dry_run: If True, log actions without making API calls

    Returns:
        0 on success, 1 on error
    """
    # 1. Load the outbound plan
    plan = self.load_json(plan_path)
    if not plan:
        logger.error(f"Failed to load outbound plan: {plan_path}")
        return 1

    # 2. Load rate limits for today
    rate_limits = self.load_rate_limits()

    # 3. Execute each target's actions
    for target in plan.get("targets", []):
        handle = target["handle"]

        if target.get("skip"):
            logger.info(f"Skipping {handle}: {target.get('skip_reason', 'no reason')}")
            continue

        # Use user_id from plan (already resolved by publisher_outbound_data.py)
        user_id = target.get("user_id")
        if not user_id:
            logger.warning(f"No user_id in plan for {handle}, skipping")
            continue

        # Like selected tweets
        for tweet_id in target.get("tweets_to_like", []):
            if not self.check_rate_limit(rate_limits, "likes", 30):
                logger.warning("Daily like limit reached")
                break
            if not dry_run:
                success = self.write_client.like_tweet(tweet_id)
                if success:
                    self.increment_rate_limit(rate_limits, "likes")
                self._delay_random()
            self._log_outbound_action("like", account, handle, tweet_id)

        # Reply to selected tweet
        reply_info = target.get("reply_to")
        if reply_info and self.check_rate_limit(rate_limits, "replies", 10):
            reply_text = f"@{handle.lstrip('@')} {reply_info['reply_text']}"
            if not dry_run:
                success = self.write_client.create_tweet(
                    text=reply_text,
                    reply_to=reply_info["tweet_id"]
                )
                if success:
                    self.increment_rate_limit(rate_limits, "replies")
                self._delay_random()
            self._log_outbound_action("reply", account, handle, reply_info["tweet_id"])

        # Follow
        if target.get("follow", False) and self.check_rate_limit(rate_limits, "follows", 5):
            if not dry_run:
                success = self.write_client.follow_user(user_id)
                if success:
                    self.increment_rate_limit(rate_limits, "follows")
                self._delay_random()
            self._log_outbound_action("follow", account, handle)

    # 4. Save rate limits and outbound log
    self.save_rate_limits(rate_limits)
    return 0
```

```python
#!/usr/bin/env python3
"""Fetch target account data for Claude Publisher smart outbound planning.

Usage:
    python3 scripts/publisher_outbound_data.py --account EN --targets "@handle1,@handle2"

Output: JSON printed to stdout with target account info and recent tweets.
"""

import argparse
import json
import sys
from x_api import XApiClient, load_bearer_token

class OutboundDataFetcher:
    """Fetches target account info and recent tweets for Claude analysis."""

    def __init__(self):
        """Initialize with read-only X API client.

        Uses the shared bearer token (read-only access) — same as
        scout.py and analyst.py. Account parameter is not needed for
        read-only operations; the bearer token works across all accounts.
        """
        bearer = load_bearer_token()
        self.client = XApiClient(bearer)

    def fetch_target(self, handle: str) -> dict:
        """Fetch one target account's info and recent tweets.

        Steps:
        1. Resolve handle to user_id
        2. Fetch user info (followers, bio)
        3. Fetch 5 most recent tweets with full text and public_metrics

        Args:
            handle: X handle (with or without @)

        Returns:
            Dict with handle, user_id, followers, bio, recent_tweets[]
        """
        handle = handle.lstrip("@")

        # Resolve user
        user_info = self.client.resolve_user_id(handle)
        if not user_info:
            return {"handle": handle, "error": "not_found"}

        # Fetch recent tweets
        tweets = self.client.get_user_timeline(
            user_id=user_info["id"],
            max_results=5,
            tweet_fields=["text", "public_metrics", "created_at", "entities"]
        )

        return {
            "handle": f"@{handle}",
            "user_id": user_info["id"],
            "followers": user_info.get("public_metrics", {}).get("followers_count", 0),
            "bio": user_info.get("description", ""),
            "recent_tweets": [
                {
                    "tweet_id": t["id"],
                    "text": t["text"],
                    "likes": t.get("public_metrics", {}).get("like_count", 0),
                    "retweets": t.get("public_metrics", {}).get("retweet_count", 0),
                    "replies": t.get("public_metrics", {}).get("reply_count", 0),
                    "created_at": t.get("created_at", "")
                }
                for t in tweets
            ]
        }

    def fetch_all(self, targets: list[str]) -> dict:
        """Fetch data for all target accounts.

        Args:
            targets: List of X handles

        Returns:
            Dict with targets[] array
        """
        results = []
        for handle in targets:
            data = self.fetch_target(handle)
            results.append(data)
        return {"targets": results}


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--account", required=True, choices=["EN", "JP"])
    parser.add_argument("--targets", required=True, help="Comma-separated handles")
    args = parser.parse_args()

    targets = [t.strip() for t in args.targets.split(",")]
    fetcher = OutboundDataFetcher()
    result = fetcher.fetch_all(targets)

    print(json.dumps(result, indent=2, ensure_ascii=False))


if __name__ == "__main__":
    main()
```

## 10. Edge Cases & Gotchas

### 10.1 Context Window (457KB Scout Report)

| Case | Expected Behavior |
|---|---|
| Scout report exceeds Claude context | `--compact` flag strips `recent_posts`, reducing to ~30KB |
| `--compact` output still too large | Scout subagent reads competitor data in batches (not expected with 41 competitors) |
| Raw file needed for deep analysis | Scout reads full `data/scout_raw_{date}.json` for specific competitors by handle |

### 10.2 Claude Subagent JSON Failures

| Case | Expected Behavior |
|---|---|
| Claude wraps JSON in markdown code fences | Marc's H3 retry specifies "no code fences" explicitly |
| Claude produces partial JSON (truncated) | `validate.py` catches JSON parse error; Marc retries once |
| Claude adds commentary before/after JSON | `validate.py` catches non-JSON prefix; Marc retries with "first character must be `{`" |

### 10.3 Smart Outbound Approval Timing

| Case | Expected Behavior |
|---|---|
| Operator doesn't approve within 2 hours | Marc logs timeout, skips outbound for the day |
| Operator rejects plan | Marc logs rejection, falls back to legacy outbound |
| Operator approves after rate limit window | Marc checks rate limits before executing; may have reduced capacity |

### 10.4 Fallback Behavior

| Case | Expected Behavior |
|---|---|
| Scout subagent fails after retry | Marc runs `python3 scripts/scout.py` (no `--raw`, no analysis) |
| Publisher subagent fails after retry | Marc runs `python3 scripts/publisher.py outbound --account {account}` |
| Analyst subagent fails after retry | Marc reads metrics summaries, applies anomaly formula manually, composes Telegram message |
| All three subagents fail | System degrades to Phase 4 behavior — fully functional but no intelligence layer |

### 10.5 Backward Compatibility

| Case | Expected Behavior |
|---|---|
| Strategist encounters `analysis` section in scout report | Ignores it — reads only existing fields (`competitors[]`, `market_comparison`, etc.) |
| `daily_report` file missing (first run) | Analyst's trend comparison skips gracefully; reports "no previous data" |
| `outbound_log` missing (no outbound run today) | Analyst reports zero outbound activity |
| Content plan has no `ab_test_variant` on posts | Analyst reports `verdict: "no_data"` for A/B test |
